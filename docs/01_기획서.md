# StockPulse 기획서

> 실시간 투자 인사이트 · 모의투자 · 종목 커뮤니티 플랫폼
> Kotlin + Spring Boot MSA 기반 고가용성 실시간 시스템

---

## 1. 프로젝트 개요

### 1.1 배경

이전 프로젝트 **SafeAlert**(재난 알림 구독 MSA)에서 MSA·Kafka·Kubernetes·관측 가능성 등 분산 시스템의 기본기를 구축했다. 그러나 SafeAlert는 *저빈도 배치성 데이터(1시간 폴링)*와 *단순 단방향 푸시* 중심이라, 대규모 실시간 시스템이 진짜로 어려워지는 지점 — 고빈도 스트리밍, 다중 서비스에 걸친 트랜잭션, 양방향 실시간 — 을 충분히 다루지 못했다.

**StockPulse**는 같은 기술 스택을 **의도적으로 더 어려운 도메인(실시간 주식)**에 적용해, 분산 시스템 역량을 한 단계 끌어올리는 것을 목표로 한다.

### 1.2 목적

1. **요구 역량 전부 증명**: 고가용성·확장성, 대용량 실시간 트래픽, 장애 경험/해결, Redis·Kafka, MQ 비동기, 이벤트 기반 아키텍처, 분산 트랜잭션/Saga/Outbox, timeout/retry/circuit breaker, 관측(로그·메트릭·트레이싱), Prometheus·Grafana.
2. **실사용 가능한 가용성 있는 웹 서비스**: 데모로 끝나지 않고 실제 접속해 사용할 수 있는 서비스로 배포.

### 1.3 한 줄 정의

> "보고(인사이트) → 토론하고(커뮤니티) → 행동하는(모의투자)" 하나의 루프로 연결된 실시간 주식 플랫폼.

## 2. SafeAlert와의 차별점 (의도적으로 더 어렵게)

| 축 | SafeAlert | StockPulse |
|----|-----------|------------|
| 데이터 유입 | 공공 API **1시간 폴링** (저빈도 배치) | KIS **실시간 WebSocket 고빈도 틱 스트리밍** |
| 핵심 트랜잭션 | 구독 변경 → 단순 Outbox 1-step | 모의투자 주문 → **다중 서비스 Saga + 보상 트랜잭션** |
| 실시간 방향 | 단방향 푸시(알림) | **양방향**(시세 푸시 + 종목별 실시간 채팅) |
| 데이터 처리 | 단일 소스 분류·필터 | **멀티 소스 융합**(시세+뉴스+재무 → 인사이트 스코어) |
| 스트림 처리 | 단순 `@KafkaListener` 소비 | **Kafka Streams** 윈도우 집계로 지표 계산 |
| 저장소 | PostgreSQL·MongoDB | + **TimescaleDB**(시계열 틱·캔들) |
| 언어/비동기 | Java, 스레드 | **Kotlin + Coroutines** |
| 배포 | 수동 `kubectl apply` | **GitHub Actions CI/CD + Helm** |
| 관측 | ELK + Jaeger | **Loki + Tempo**(all-Grafana 경량 스택) |
| 클라이언트 | React 웹 | React 웹 + **React Native(Expo) 앱 + FCM 푸시** |

## 3. 핵심 기능 (3대 기둥 + 실용 기능)

### 3.1 실시간 투자 인사이트 (전망 카드)

주가가 "오를지 맞히는" 예측이 아니라, **공개 데이터를 종합해 설명 가능한 근거를 보여주는** 정보 제공 기능. 종목별로 **4개 축**을 점수화한다. "전망(방향)"과 "가격 적정성(밸류에이션)"을 분리해 둘 다 명확히 전달한다.

- **모멘텀**: 실시간 시세 스트림 기반 — 기간 수익률, 거래량 급증, 이동평균 정렬, RSI 등 기술적 지표
- **실적**: DART 재무 — 영업이익·매출 성장률(YoY), 흑자 전환 등 (회사가 좋아지고 있나)
- **밸류에이션**: PER·PBR·PEG·배당수익률을 **업종 평균·과거 밴드 대비 상대 위치**로 평가 (지금 가격이 싼가/비싼가)
- **뉴스 호재/악재**: 네이버 뉴스 수집 → 감성/키워드 분류

→ 종합 결과를 카드로 제시. 예시:

```
[삼성전자]  종합 전망: 긍정 78점
  ├ 모멘텀   강세 (5일 +4.2%, 거래량 1.8배, 골든크로스)
  ├ 실적     양호 (영업이익 YoY +12%)
  ├ 밸류에이션 다소 저평가 (PER 11.3 · 업종평균 15.2↓ · 5년밴드 하위 30%)
  └ 뉴스     호재 다수 (HBM 공급 확대 보도 3건)
```

각 근거는 펼쳐서 원천 데이터(지표 값·뉴스 원문 링크)를 확인할 수 있다.

**정확성 원칙**: 미래 가격 예측은 본질적으로 불가능하므로, *정확한 예측*이 아니라 **정확한 데이터·계산 + 설명 가능성**을 목표로 한다. ① 공식 데이터원(DART·KIS) ② 지표 계산 테스트 검증 ③ 단정 대신 상대 위치 제시. 추가로 **백테스트**로 "이 점수가 과거에 실제로 의미 있었는지"를 검증해 적중률을 투명하게 공개한다(예: 긍정 75점↑ 종목의 30일 후 평균 수익률). 미래를 맞힌다고 주장하는 대신 **과거 성적표를 공개**하는 정직한 방식.

### 3.2 종목별 실시간 커뮤니티

- 종목별 토론방, 실시간 채팅, 게시글·댓글
- 대용량 동시접속을 견디는 WebSocket fan-out (Redis Pub/Sub로 다중 인스턴스 메시지 전파)

### 3.3 모의투자 (분산 트랜잭션 핵심)

- 가상 잔고로 매수/매도 주문 → 현재 시세로 체결
- **주문(Trading) → 잔고 예약(Account) → 체결 → 보유 갱신(Portfolio) → 원장 확정**이 **3개 서비스에 걸친 분산 트랜잭션(Saga)**, 실패 시 보상으로 롤백
- 수익률 계산, 실시간 수익률 랭킹
- ※ 실제 금전 거래는 법적으로 불가(자본시장법) → 거래는 전부 가상으로 유지
- **포지셔닝**: "초보 투자자를 위한 인사이트 실습 도구". 전망 카드를 보고 모의로 매매 → 결과로 학습하는 루프를 완성하는 기능 (단독 샌드박스가 아니라 인사이트와 연계)

### 3.4 실용 기능 (실사용·앱 가치)

- **워치리스트/관심종목**: 내 종목을 모아 시세·인사이트·뉴스 추적 (개인화)
- **FCM 실시간 알림**: 목표가 도달·급등락·뉴스 호재 시 앱 푸시 — 앱을 켜지 않아도 가치를 주는 핵심 기능
- **앱 확장**: React Native(Expo)로 iOS/Android 앱 제공, FCM 푸시 수신

## 5. 시스템 아키텍처 개요

Kotlin + Spring Boot MSA. SafeAlert의 Gateway/Auth/관측 패턴 재사용.

**수집·처리 파이프라인**
- **Market Collector** — KIS 실시간 WebSocket 시세(국내+해외) → Kafka `market.tick`. CB. 리플레이/증폭 모드 내장(부하용).
- **News Collector** — 네이버 검색 API 주기 수집 → `news.raw`. CB.
- **Fundamentals Collector** — DART Open API 재무/공시 주기 수집 → `fundamentals.raw`. CB.
- **Insight Service** — 3 스트림 소비 → 모멘텀·실적·밸류에이션·뉴스 **4축 스코어** → 전망 카드 산출 + 백테스트 → 저장 + `insight.updated`. Replica 확장.

**모의투자 (Saga 도메인)**
- **Trading Service** — 주문 접수, Saga 오케스트레이션, Outbox 발행.
- **Account Service** — 가상 잔고 예약·확정·해제, 원장.
- **Portfolio Service** — 보유종목·평가금액·수익률.
- **Ranking Service** — Redis Sorted Set 수익률 랭킹.

**실시간·사용자**
- **Realtime Gateway (WebSocket)** — 시세 푸시 + 종목 채팅 + 인사이트 푸시. STOMP + Redis Pub/Sub fan-out.
- **Community Service** — 토론방·게시글·댓글. Outbox → 알림.
- **Notification/Alert Service** — 알림 규칙(목표가·급등락) 평가, **FCM 푸시**.
- **API Gateway** — Spring Cloud Gateway, JWT, Redis Rate Limit.
- **Auth/User Service** — JWT Access/Refresh, OAuth2, **워치리스트/관심종목**.

> 총 13개 서비스. 상세 구성도는 [02_시스템아키텍처.md](02_시스템아키텍처.md) 참고.

**데이터 저장소**
- PostgreSQL: user, account/portfolio, orders/ledger, community
- TimescaleDB: 시계열 틱·캔들 (Redis는 최신가+단기 윈도우 캐시)
- MongoDB: 뉴스·인사이트·백테스트 결과 문서
- Redis: 캐시·랭킹·Rate Limit·Pub/Sub·세션

**관측**: Prometheus·Grafana·Loki(로그)·Tempo(트레이스, OpenTelemetry)

## 6. 모의투자 Saga 흐름

매수 주문 예시 (오케스트레이션 기반, 보상 포함):

```
OrderRequested (Trading)
  → ReserveBalance (Account)        실패 시 → 주문 거절
  → ExecuteOrder (Trading, 현재 시세 체결)   실패 시 → ReleaseBalance(보상)
  → UpdateHoldings (Portfolio)      실패 시 → ReleaseBalance + CancelExecution(보상)
  → ConfirmOrder (Trading) + WriteLedger
  → OrderCompleted 이벤트 → Ranking 평가 갱신
```

- Trading이 Saga 상태 머신을 보유, 각 단계는 Kafka 이벤트로 연결.
- **Outbox**: 각 서비스가 로컬 DB 트랜잭션 + Outbox 테이블에 이벤트를 함께 기록 후 발행 → Kafka 일시 장애에도 유실 없음.
- 멱등성 키로 중복 체결 방지.

## 7. 데이터 소스 및 한계

| 소스 | 용도 | 접근성 | 한계 / 대응 |
|------|------|--------|------------|
| 한국투자증권 KIS API | 국내·해외 실시간 시세, 모의투자 | 무료, 계좌+앱키. 국내장(09~15:30)+미국장(한국 23:30~06:00) → 데모 시간대 넓음. 모의투자 전용 도메인 제공 | 실시간 동시 등록 ~40종목 제한 → 실데이터는 제한 수집, 대용량은 시세 리플레이/증폭기로 합성 |
| 네이버 검색 API | 종목 뉴스 | 무료, Client ID/Secret | 일일 호출 쿼터 → 종목 우선순위 큐로 폴링 주기 조절 |
| DART Open API | 재무·공시 | 무료, 인증키 | 분기성 데이터 → 일배치 수집으로 충분 |

> 원칙: Phase 0에서 세 API 모두 **키 발급 + 실제 1회 호출 성공**을 확인한 뒤 수집기 코딩을 시작한다.

## 8. 기술 스택

- **Backend**: Kotlin, **Coroutines**, Spring Boot 3.x, Spring Cloud Gateway, Resilience4j
- **Messaging / Streaming**: Apache Kafka (KRaft), **Kafka Streams**(윈도우 집계 지표 계산)
- **Store**: PostgreSQL, **TimescaleDB**(시계열 틱·캔들), MongoDB, Redis 7
- **Frontend(웹)**: React 18 + Vite + Tailwind + **TradingView Lightweight Charts**
- **App**: **React Native (Expo)** + **FCM**(푸시)
- **Infra / 배포**: Docker, Kubernetes(HPA), **Helm**, **GitHub Actions CI/CD**
- **Observability**: Prometheus, Grafana, **Loki**(로그), **Tempo**(트레이스, OpenTelemetry)
- **Test**: JUnit5(Kotlin), **Testcontainers**(통합), k6(부하)
- **장애 테스트(Phase 7)**: **Chaos Mesh**

> 위 굵은 항목은 SafeAlert와 차별화를 위해 추가 확정한 스택이다. 선택 근거는 [07_기술선택이유.md](07_기술선택이유.md) 참고.

## 10. 규제 / 면책

- 본 서비스는 **투자자문이 아니라 공개 데이터 종합 정보 제공**이다.
- 모든 거래는 **가상(모의)**이며 실제 금전 거래가 없다.
- UI에 면책 문구를 명시한다.
